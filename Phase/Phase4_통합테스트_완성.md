## 통합 테스트 시나리오

### 10시간 빨리감기 테스트
워크플로 A(시뮬레이터)를 10번 클릭하여 08:00~17:00 하루 전체를 시뮬레이션.
각 시간마다 워크플로 B(모니터)가 자동 실행되어 이상 탐지 → AI 해석 → 이메일 발송.

### 검증 항목
| 시간 | 기대 동작 | 검증 포인트 |
|---|---|---|
| 08:00 | 첫 시간, 비교 대상 없음 | summary만 저장, 이상 없음 |
| 09:00 | 도장 라인 불량률 급등 시작 | 불량률 정상의 8배 탐지, 심각 |
| 10:00 | 조립 2라인 작업자 교체 | 복합 이상 (생산↓+불량↑) 탐지 |
| 11:00 | CNC 1호기 설비 고장 시작 | 생산량 급감 탐지, AI 해석 호출 |
| 12:00 | CNC 연속 0, 도장 개선 | consecutive_zero + 반복/악화 분류 |
| 13:00 | CNC 연속 0, 사출 금형 이상 | 동시 다발 AI 일괄 처리 검증 |
| 14:00 | CNC 시운전 복구 | 이상 해소 확인 |
| 15:00 | 프레스 자재 대기 시작 | 가동률 급락 탐지 |
| 16:00 | 프레스 계속 | 30분 반복 알림 (에스컬레이션) |
| 17:00 | 프레스 일부 입고 | 상황 변화 + 일일 리포트 준비 |

## 워크플로 C: 주간 백업

### 테스트 내용
- 매주 월요일 07:50 실행 시뮬레이션
- production_week.xlsx → /backup/ 폴더에 `production_YYYYMMDD_YYYYMMDD.xlsx`로 복사
- 원본 삭제 (08:00에 외부에서 새 파일 생성 가정)
- 지난 주 daily_summary 생성 (일별 팀별 집계)

### 검증 포인트
- 백업 파일명에 기간이 정확히 표시되는지
- daily_summary 집계값 정확성 (총실적/총계획 재계산, 가중평균 가동률)

## 워크플로 D: 일일 리포트 (AI Agent)

### 테스트 내용
- 매일 08:00 실행 시뮬레이션
- AI Agent가 도구(Tool Use)를 활용하여 자율적으로 데이터 조회 및 분석
- Agent의 리포트 깊이 자율 조절 동작 검증
- 안전장치(폴백, 호출 제한) 동작 검증

### AI Agent 동작 검증

| 테스트 시나리오 | 기대 동작 | 검증 포인트 |
|---|---|---|
| 조용한 날 (낮음 2건) | Agent가 도구 1~2회 호출, 간단 리포트 | report_level="간단", 도구 호출 최소화 |
| 보통 날 (심각 1건 + 중간 2건) | Agent가 도구 3~4회, 이력 비교 수행 | 지난주 대비 추세 분석 포함 |
| 심각한 날 (심각 3건+) | Agent가 도구 5~6회, 상세 리포트 | report_level="상세", cc_management=true |
| 악화 추세 (같은 라인 반복) | Agent가 7일 이력 자율 조회 | 에스컬레이션 패턴 감지 |

### 안전장치 테스트

| 테스트 | 방법 | 기대 결과 |
|---|---|---|
| 도구 호출 제한 | 대량 이상 데이터 투입 | 8회 초과 시 루프 종료, 리포트 생성 |
| API 장애 폴백 | Claude API 키 임시 제거 | 기존 고정 포맷 리포트 발송 |
| 파싱 실패 | - | Agent 텍스트 그대로 이메일 포함 |
| 도구 실패 | OneDrive 파일 임시 삭제 | Agent가 "조회 실패" 인지 후 가능한 범위 작성 |

### 리포트 구성 (Agent 자율)
- 전날 총 이상 건수 (심각 N건, 중간 N건, 낮음 N건)
- 심각 상세 (각 건의 AI 분석 포함)
- 중간 요약
- 낮음 리스트
- 반복/악화 하이라이트
- 추세 분석 (심각 건이 있을 때 Agent가 자율 추가)
- AI 종합 코멘트 (개선 제언)
- Power BI 대시보드 링크

## 데이터 수명주기 관리

### 구조
```
현재 주: production_week.xlsx (최대 720행) ← Power BI 직접 연결
지난 주~: /backup/ 파일 단위 보관 ← Power BI 연결 없음
장기:    daily_summary 탭 누적 (월 ~90행) ← Power BI 페이지 3
```

### 설계 결정 사항
- **파일 통째로 이동**: 행 단위 회전보다 단순하고 안전
- **주간 파일(720행)**: 하루 단위(여러 파일 참조) vs 월 단위(3600행 과도) 사이의 균형점
- **데모: 무기한 보관** / 실무: 3~6개월 후 아카이브
